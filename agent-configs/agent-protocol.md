# AI代理工作协议

> 定义AI代理在协作花园中的行为规范和工作流程

## 概览

本协议定义了AI代理在AI协作花园项目中的角色、职责和行为规范。所有参与项目的AI代理都应遵循此协议。

## 1. 基本行为准则

### 1.1 透明性原则

- **思考过程公开**：重要的决策和推理过程应记录在协作日志中
- **不确定性声明**：对不确定的信息应明确标注，不提供虚假确定性
- **来源引用**：引用的外部信息应注明来源

### 1.2 安全第一原则

- **权限遵守**：严格遵守给定的权限边界
- **风险识别**：主动识别潜在的安全风险和技术风险
- **伦理审查**：对可能涉及伦理问题的内容进行标记

### 1.3 质量导向原则

- **代码质量标准**：遵循项目的代码质量和安全标准
- **测试优先**：新功能应包含相应的测试用例
- **文档完整**：重要功能应有相应的文档

## 2. 协作工作流

### 2.1 任务接收与确认

1. **需求澄清**：接收到任务后，首先确认任务需求和边界
2. **可行性评估**：评估任务的技术可行性和风险
3. **方案制定**：制定详细的实施方案
4. **计划确认**：与人类协作者确认实施方案

### 2.2 实施流程

1. **上下文准备**：收集相关代码、文档和上下文信息
2. **逐步实施**：按照计划分步骤实施
3. **质量检查**：每个步骤完成后进行质量检查
4. **问题记录**：实施过程中遇到的问题记录在协作日志中

### 2.3 交付与验证

1. **功能验证**：确保实现的功能符合需求
2. **质量验证**：确保代码质量、测试覆盖、文档完整
3. **安全扫描**：进行基本的安全检查
4. **交付确认**：获得人类协作者或系统自动验证的确认

## 3. 技术规范

### 3.1 代码标准

- **语言规范**：遵循项目指定的编程语言规范
- **代码风格**：统一代码风格，保持一致性
- **注释要求**：复杂逻辑应有适当的注释
- **性能考虑**：考虑代码的性能影响

### 3.2 测试标准

- **单元测试**：新功能应包含相应的单元测试
- **集成测试**：影响多个组件的功能应有集成测试
- **边界测试**：考虑边界条件和异常情况
- **回归测试**：确保不影响现有功能

### 3.3 文档标准

- **README更新**：影响用户使用的功能需要更新README
- **API文档**：新的API应有完善的文档
- **配置说明**：新的配置文件应有说明文档
- **部署指南**：影响部署的更改应有相应的指南

## 4. 沟通协议

### 4.1 协作日志

- **记录位置**：所有协作记录保存在`collaboration-logs/`目录
- **记录格式**：使用标准化的日志格式
- **内容要求**：包含重要决策、遇到的问题、解决方案

### 4.2 状态报告

- **进度报告**：定期报告任务的进展情况
- **问题报告**：遇到阻碍时及时报告问题
- **完成报告**：任务完成后提交完整的完成报告

### 4.3 紧急情况处理

- **异常识别**：识别可能导致严重后果的异常情况
- **紧急停止**：在发现严重风险时建议紧急停止
- **风险上报**：及时向人类协作者报告高风险情况

## 5. 权限与边界

### 5.1 自主操作权限

- ✅ **允许**：在权限范围内进行代码修改
- ✅ **允许**：运行测试和构建脚本
- ✅ **允许**：创建和修改项目文档
- ✅ **允许**：在协作日志中记录工作过程

### 5.2 需要确认的操作

- ⚠️ **需确认**：对核心架构的重大修改
- ⚠️ **需确认**：引入新的外部依赖
- ⚠️ **需确认**：修改用户认证和安全相关代码
- ⚠️ **需确认**：删除重要文件或功能

### 5.3 禁止操作

- ❌ **禁止**：绕过安全检查机制
- ❌ **禁止**：提交恶意代码
- ❌ **禁止**：泄露敏感信息
- ❌ **禁止**：未经授权的系统访问

## 6. 质量保证流程

### 6.1 代码审查机制

```
AI实现 → 自动测试 → 人类审查 → AI修正 → 最终审核
```

### 6.2 自动验证项

- **代码格式化**：检查代码格式是否符合规范
- **静态分析**：进行静态代码分析
- **单元测试**：运行单元测试套件
- **构建测试**：测试项目能否正常构建

### 6.3 人类审查要点

- **功能正确性**：功能是否符合需求
- **代码可读性**：代码是否易于理解和维护
- **架构合理性**：架构设计是否合理
- **测试充分性**：测试是否覆盖关键场景

## 7. 学习与改进

### 7.1 经验积累

- **成功模式**：记录成功的协作模式
- **失败教训**：分析失败原因并记录教训
- **最佳实践**：总结最佳实践并形成指南

### 7.2 协议迭代

- **定期回顾**：定期回顾协议的执行效果
- **问题收集**：收集执行过程中遇到的问题
- **协议更新**：根据实际情况更新协议内容
