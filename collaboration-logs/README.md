# 协作日志目录

> 记录AI与人类协同工作的过程和思考

## 目的

协作日志旨在完整记录AI代理与人类开发者的协同工作过程，包括：

- **思考过程**：AI的推理路径和决策依据
- **沟通记录**：人类与AI的对话和交流
- **实施步骤**：具体的实施步骤和方法
- **问题解决**：遇到的问题和解决方案
- **经验总结**：从协作中学到的经验和教训

## 日志格式

### 文件名格式

```
YYYY-MM-DD-项目名-任务描述.md
```

示例：

```
2026-03-20-weather-cli-天气图标对齐优化.md
```

### 日志内容结构

每个日志文件应包含以下部分：

```markdown
# [任务名称]

**日期**：YYYY-MM-DD  
**参与者**：[人类开发者]/[AI代理]  
**任务状态**：✅ 完成 / ⚠️ 进行中 / ❌ 取消

## 背景与目标

[描述任务的背景和预期目标]

## 实施计划

[描述初步的实施计划]

## 实施过程

### 阶段1：[阶段名称]

[详细记录该阶段的实施过程]

### 遇到的问题

- 问题1：[描述]
  - 解决方案：[描述]
  - 结果：[描述]

## 交付成果

- [交付项1]
- [交付项2]

## 质量验证

- [验证项1]
- [验证项2]

## 经验总结

### 成功因素

- [因素1]
- [因素2]

### 改进建议

- [建议1]
- [建议2]

## 附件

[相关代码片段、截图、日志文件等]
```

## 日志管理

### 新建日志

1. 在开始新任务时创建新的日志文件
2. 使用标准文件名格式
3. 填写基本信息

### 更新日志

1. 任务进行中定期更新日志
2. 记录重要的决策和进展
3. 遇到问题时及时记录

### 归档日志

1. 任务完成后标记状态为"完成"
2. 添加经验总结和改进建议
3. 确保所有相关信息都已记录

## 最佳实践

### 记录频率

- **重要决策**：立即记录
- **进展更新**：每完成一个重要步骤后记录
- **问题解决**：问题解决后立即记录

### 记录内容

- **详细但不冗长**：记录关键信息，避免无关细节
- **客观中立**：客观记录事实，避免主观评价
- **可追溯**：确保信息可追溯和验证

### 隐私与安全

- **不含敏感信息**：不记录密码、API密钥等敏感信息
- **公开适当信息**：确保日志内容适合公开分享
- **数据保护**：遵守数据保护相关法律法规

## 注意事项

1. **完整性**：确保日志记录完整的协作过程
2. **可读性**：保持良好的可读性，便于他人理解
3. **价值性**：记录有价值的信息和经验
4. **持续性**：坚持记录，形成完整的协作历史
