## 实施过程

### 阶段1：问题分析与方案设计

**问题分析结果**：

1. **香港/澳门冲突**：cityAdcodes映射中香港/澳门与Open-Meteo API冲突，导致无法正确查询
2. **图标对齐**：中文字符宽度影响图标对齐，需要像素级宽度计算
3. **日期显示**：3天天气预报日期计算错误，星期几显示错误
4. **行政区划**：现有adcode查询仅支持到市一级，无法查询县区
5. **API配置**：首次使用需要手动配置.env文件，用户体验不佳

**解决方案**：

1. 从cityAdcodes移除香港/澳门，完全使用Open-Meteo API
2. 引入`string-width`库计算字符串像素宽度，实现完美对齐
3. 修正日期计算逻辑，调整星期几计算方式
4. 实现`getExtendedCityAdcode()`和`getAdcodeByGeocode()`支持分层查询
5. 实现`ensureGaodeApiKey()`函数，首次使用时引导配置

## 遇到的问题

### 问题1：中文字符宽度计算不准确

**描述**：普通空格无法实现完美对齐，中文字符宽度为2个英文字符宽度
**解决方案**：引入`string-width`库精确计算字符串显示宽度
**结果**：实现像素级完美对齐

### 问题2：高德地理编码API限制

**描述**：某些县区名称在高德API中返回多个结果，需要准确匹配
**解决方案**：添加结果过滤和选择逻辑，优先匹配精准结果
**结果**：提高县区查询的准确性

### 问题3：API密钥配置流程中断

**描述**：配置过程中用户可能取消或输入错误
**解决方案**：添加重试机制和输入验证
**结果**：提升配置流程的健壮性

#### 图标格式化函数

```typescript
function formatIcon(icon: string): string {
  const width = stringWidth(icon);
  const targetWidth = 4; // 假设目标宽度为4个英文字符宽度
  if (width >= targetWidth) return icon;
  const spaces = ' '.repeat(targetWidth - width);
  return icon + spaces;
}
```
